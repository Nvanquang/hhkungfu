## Ghi Chú Thiết Kế

### Trang Chọn Gói VIP
- Card gói "Phổ biến nhất" (3 Tháng): border primary, scale nhẹ hơn 2 card còn lại, badge ⭐
- Tiết kiệm % tính dựa `original_price` từ DB (hiện nếu original_price != null)
- FAQ dạng accordion, click tiêu đề mở/đóng nội dung

### Trang Thanh Toán
- Chỉ 2 phương thức: VNPay và MoMo — radio button, click 1 cái bỏ cái kia
- "Thanh toán ngay" → POST `/subscriptions/initiate` → nhận `paymentUrl` → `window.location.href = paymentUrl`
- Nếu `emailVerified = false`: nút bị disabled + warning banner vàng
- Hiện thông tin gia hạn nếu user đang có VIP (cộng dồn expires_at)

### Trang Kết Quả
- URL: `/vip/result?orderId=ORD-xxx`
- Load: `GET /payments/result/ORD-xxx` → poll mỗi 3 giây nếu `status = PENDING`
- Stop poll khi `status ≠ PENDING` hoặc sau 60 giây
- 3 states: `PENDING` (spinner) · `PAID` (success) · `FAILED/CANCELLED` (error)
- "Bắt đầu xem ngay" → về trang chủ hoặc trang anime đang xem dở

### Redirect Flow
```
User chọn gói
  → POST /subscriptions/initiate
  → { paymentUrl, orderId }
  → redirect to VNPay/MoMo
  → (user thanh toán trên VNPay/MoMo)
  → VNPay/MoMo redirect về /vip/result?orderId=xxx
  → FE poll /payments/result/:orderId
  → Hiện kết quả
```
